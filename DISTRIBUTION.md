# Distribution Policy

WakeFromFar source code is open and available under the repository license.

## Official distributions

The official maintained binaries are the ones published by the project/publisher through the official store listings:

- Apple App Store for iPhone
- Google Play for Android

Self-built binaries, third-party forks, and redistributed builds are not official unless explicitly stated by the project maintainers.

## Forks and redistribution

Forks may use, modify, and redistribute the source code according to the repository license, but they must not imply:

- official project status
- maintainer endorsement
- affiliation with the official store releases

If you redistribute modified builds, rename them and replace branding assets so users can clearly distinguish them from the official releases.

## Source builds

The repository is intended to remain buildable and self-hostable outside the app stores. Source-built usage and self-hosted deployment are valid project use cases.

## Store policy alignment

- Official Android store releases use Google Play billing/policy requirements where applicable.
- Official iPhone store releases follow Apple's App Store distribution rules.
- Store policy requirements for official binaries do not prevent source availability of the code itself.
